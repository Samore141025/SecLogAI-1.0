# Contributing to SecLogAI

First off, thank you for considering contributing to SecLogAI! It's people like you that make SecLogAI such a great tool.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs

- **Check the FAQ** to see if the issue has already been addressed.
- **Search existing issues** to see if the bug has already been reported.
- **Open a new issue**, providing as much context as possible (browser version, OS, steps to reproduce).

### Suggesting Enhancements

- **Open a new issue** with the tag "enhancement".
- Describe the feature and why it would be useful.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies** with `npm install`.
3. **Make your changes**.
4. **Ensure the code lints** with `npm run lint`.
5. **Submit a Pull Request** with a clear description of the changes.

## Style Guide

- Use TypeScript for all new code.
- Follow the existing Tailwind CSS patterns for styling.
- Ensure all new components are responsive.
- Add comments to complex logic.

## Security

If you discover a security vulnerability, please do NOT open an issue. Instead, follow the instructions in [SECURITY.md](SECURITY.md).
